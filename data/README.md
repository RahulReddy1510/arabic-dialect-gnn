# Data Directory

This directory contains the full data pipeline: MADAR corpus preprocessing,
CAMeL Tools phoneme extraction, graph construction, and the PyTorch Geometric
dataset class.

---

## Dataset: MADAR Corpus

This project uses the **MADAR corpus** (Multi-Arabic Dialect Applications and
Resources), a parallel sentence-level corpus covering 25 Arabic-speaking cities.

**Citation:**

```
Bouamor, H., Habash, N., Salameh, M., Zaghouani, W., Rambow, O., Abdulmageed, M.,
Mubarak, H., Tomeh, N., Abbas, O., & Samih, Y. (2018). The MADAR Arabic Dialect
Corpus and Lexicon. Proceedings of LREC 2018.
```

### Accessing MADAR

MADAR requires signing a data usage agreement. Request access here:

- **MADAR 26-city travel dataset**: https://camel.abudhabi.nyu.edu/madar/
- Complete the form and an institutional email is required.
- Data is typically delivered within 1–3 business days.

Once you have the data, place the TSV files here:

```
data/
├── raw/
│   ├── MADAR-Corpus-26-train.tsv
│   ├── MADAR-Corpus-26-dev.tsv
│   └── MADAR-Corpus-26-test.tsv
```

### Corpus Format

MADAR files are tab-separated with three columns:

```
sentence\tcity\tdialect
```

Example:

```
كيف حالك اليوم	Dubai	Gulf
ازيك النهارده	Cairo	Egyptian
كيفك هلق	Damascus	Levantine
```

### Dialect Grouping

We map the 25 MADAR cities into 5 dialect families:

| Dialect Group | Cities in MADAR | Training Examples (approx.) |
|---|---|---|
| Gulf | Abu Dhabi, Doha, Dubai, Kuwait, Muscat, Riyadh, Sanaa | 17,200 |
| Egyptian | Cairo, Alexandria | 17,400 |
| Levantine | Beirut, Damascus, Amman, Jerusalem | 28,400 |
| Maghrebi | Tunis, Algiers, Rabat, Tripoli | 24,000 |
| Iraqi | Baghdad, Mosul, Basra | 23,000 |

Gulf Arabic is the smallest class despite spanning the most cities, which is why
training uses inverse-frequency class weighting.

> **Note on grouping**: Calling these five clusters "dialects" is a simplification.
> Sanaani Arabic (Sanaa, Yemen) is included in "Gulf" for practical reasons but
> sits at the phonological fringe of what most linguists would call Gulf Arabic.
> The grouping follows the computational dialect ID literature, not strict
> linguistic classification.

## Pipeline Overview

```
MADAR TSV files
      │
madar_preprocessing.py
  └── clean_arabic_text()
  └── split_dataset()         → data/processed/{train,val,test}.csv
      │
camel_pipeline.py
  └── CAMeLProcessor.text_to_phonemes()
  └── text_to_phoneme_graph()  → (phonemes, edges, features)
      │
graph_construction.py
  └── phonemes_to_pyg_graph()
  └── batch_construct_graphs() → data/graphs/{train,val,test}/*.pt
      │
dataset.py
  └── ArabicDialectGraphDataset
      └── PyTorch Geometric DataLoader
```

---

## Running the Pipeline

### Step 1: Preprocess MADAR

```bash
python data/madar_preprocessing.py \
    --data_dir data/raw \
    --output_dir data/processed \
    --seed 42
```

### Step 2: Build phoneme graphs

```bash
python data/graph_construction.py \
    --processed_dir data/processed \
    --output_dir data/graphs
```

### Step 3: Verify the dataset

```python
from data.dataset import ArabicDialectGraphDataset

dataset = ArabicDialectGraphDataset(root="data/graphs", split="train")
print(f"Training graphs: {len(dataset)}")
print(f"Sample: {dataset[0]}")
```

## Without MADAR

If you don't have MADAR access, all train/eval scripts will still run using
**synthetic data** generated by the dataset class. The synthetic graphs use
random phoneme sequences with realistic size distributions. This is sufficient
for verifying the pipeline and running tests — it will not reproduce the paper results.
